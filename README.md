# All the Mods 10 — Docker Server

Docker image set for a headless [All the Mods 10](https://www.curseforge.com/minecraft/modpacks/all-the-mods-10) Minecraft server.
Forked from [W3LFARe/docker-allthemods10](https://github.com/W3LFARe/docker-allthemods10) which was designed for [Unraid](https://unraid.net) (uid 99 / gid 100).

This setup is usable more broadly and takes advantage of a persistent and portable `/data` volume.
It also uses an Dockerfile approach to reduce the need to re-run the installation but without hosting any third-party files itself.

> [!NOTE]
> This repo is meant to as a generic template for NeoForge modpacks hosted on curseforge.
> To target a different CurseForge modpack, fork this repo and edit `pack.conf` and `versions.conf` to point to the versions and zip files associated with the modpack — nothing else needs to change.

## Quick start

```bash
EULA=true make quick-start
```

Builds the images, writes `.env` (environment variables), and starts the server[^1].
The server will be reachable on port `25565`.

[^1]: Setting `EULA=true` is required — it confirms you accept [Mojang's EULA](https://www.minecraft.net/en-us/eula).

## How it works

Two images are built per version:

| Image                                 | Role                                                                         |
| ------------------------------------- | ---------------------------------------------------------------------------- |
| `evilegg/all-the-mods-data:<version>` | Init container — seeds a named volume with the pre-installed NeoForge server |
| `evilegg/all-the-mods:<version>`      | Runtime — lightweight Java + entrypoint script, mounts the seeded volume     |

On first start, `docker compose up` runs the init container once to copy the pre-built server files into the persistent volume, then starts the server container.
Subsequent starts skip the copy and go straight to launching the server.

## Running with Docker Compose

`docker-compose.yml` reads image names and tag from `.env` (generated by `make compose-env`).
Pass `EULA=true` on the command line or export it — Docker Compose will error immediately if it is unset.

## Environment variables

| Variable           | Default               | Description                         |
| ------------------ | --------------------- | ----------------------------------- |
| `EULA`             | _(required)_          | Must be `true` to start the server  |
| `JVM_OPTS`         | `-Xms2048m -Xmx4096m` | Java heap flags                     |
| `MOTD`             | _(server default)_    | Message of the day                  |
| `MAX_PLAYERS`      | `5`                   | Maximum concurrent players          |
| `ONLINE_MODE`      | `true`                | Verify players against Mojang       |
| `ALLOW_FLIGHT`     | `true`                | Allow flight in survival mode       |
| `ENABLE_WHITELIST` | `false`               | Restrict logins to the whitelist    |
| `WHITELIST_USERS`  | _(empty)_             | Comma-separated Minecraft usernames |
| `OP_USERS`         | _(empty)_             | Comma-separated operator usernames  |

## Build targets

```
EULA=true make quick-start   # build images, write .env, and start the server (one-shot setup)
make                         # build data + runtime images for the default version (local arch)
make compose-env             # write .env from current pack.conf + default version
make dist                    # build + push for linux/amd64 and linux/arm64
make 6.1                     # build a specific version for local arch
make dist-6.1                # build + push a specific version for all arches
make help                    # list all available targets
```

## Adding a new modpack version

All version metadata lives in `versions.conf` — no other file needs to change.

**1. Find the new release on CurseForge.**

Open the [ATM10 files page](https://www.curseforge.com/minecraft/modpacks/all-the-mods-10/files) and click the new release.
Note the file ID from the URL and the direct download link for `Server-Files-X.Y.zip`.
The NeoForge version is listed on the same page under _Relations → Dependencies_.

**2. Append a line to `versions.conf`.**

```
# make-target  server-version  curseforge-file-id  neoforge-version  download-url
6.2  6.2  7800000  21.1.219  https://curseforge.com/.../Server-Files-6.2.zip
```

The last non-comment line automatically becomes the new default for `make` / `make dist`.

**3. Build, update `.env`, and push.**

```bash
make 6.2           # local test build
make compose-env   # update .env to point at the new tag
make dist-6.2      # push to registry for all architectures
```

## Forking for a different NeoForge modpack

```bash
gh repo create evilegg/docker-<pack-name> --template evilegg/docker-allthemods10
cd docker-<pack-name>
```

Then edit two files:

**`pack.conf`** — pack identity:

```makefile
PACK_SLUG    := <curseforge-slug>        # from the CurseForge URL
IMAGE        := evilegg/<pack-name>
IMAGE_DATA   := evilegg/<pack-name>-data
JAVA_VERSION := 21                       # or 17 for older packs
```

**`versions.conf`** — replace the ATM10 entries with the new pack's releases:

```
# make-target  server-version  curseforge-file-id  neoforge-version  download-url
1.0  1.0  <file-id>  <neoforge-version>  https://curseforge.com/...
```

Everything else — Dockerfile, Makefile, launch.sh, docker-compose.yml — works without modification.

## Resetting the server

To wipe world data and re-seed from the image, remove the named volume:

```bash
docker compose down -v
docker compose up
```

To keep world data but force a fresh server install, delete `libraries/` inside the volume — the init container will re-copy everything on next start.

## Volumes and ports

| Mount   | Purpose                                                |
| ------- | ------------------------------------------------------ |
| `/data` | All persistent state: world saves, configs, mods, logs |

| Port    | Protocol | Purpose                |
| ------- | -------- | ---------------------- |
| `25565` | TCP      | Minecraft game traffic |

## Notes

- The server runs as uid 99 / gid 100 (`minecraft` user) inside the container.
  On non-Unraid systems, ensure the `/data` mount is writable by that uid.
- `EULA=true` is required.
  By accepting it you agree to [Mojang's EULA](https://www.minecraft.net/en-us/eula).
